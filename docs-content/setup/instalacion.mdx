Esta es la checklist de todo lo que necesitas **antes de clonar el repo** (eso lo haces en el [Quick start](/docs/setup/quick-start)). Marca cada casilla; cuando estén todas, estás listo para clonar.

<Callout type="success" title="¿Ya pasaste por Fundamentos?">
Si ya seguiste [Prepara tu compu](/docs/fundamentos/prepara-tu-compu), ya deberías tener instalados **Node, yarn, Git, Supabase CLI y Cursor** (y Homebrew si estás en Mac) — no hace falta reinstalar nada. Esta página es solo para confirmar que no te falta nada suelto (sobre todo las cuentas) antes de arrancar el Quick start.
</Callout>

Si prefieres el recorrido guiado paso a paso, ve directo a [Quick start](/docs/setup/quick-start). Esta página es la **lista de verificación** para tenerlo todo a la mano.

## 1 · Programas en tu computadora

<Callout type="info" title="¿Primera vez y no sabes cómo instalarlos?">
Los pasos exactos cambian según tu equipo. Sigue la guía por sistema operativo: [Prepara tu compu (Mac / Windows)](/docs/fundamentos/prepara-tu-compu).
</Callout>

- [ ] **Homebrew (solo Mac)** — el gestor de paquetes de Mac; facilita instalar todo lo demás:
  ```bash
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  ```
- [ ] **Node 20 o 22 LTS** — [nodejs.org](https://nodejs.org) (versión **par**/LTS). En Mac con Homebrew: `brew install node`. *(Opcional, avanzado: [nvm](https://github.com/nvm-sh/nvm) es un programa para tener varias versiones de Node y cambiar entre ellas; el repo trae un archivo `.nvmrc` que le dice cuál usar — con nvm instalado, corre `nvm use`.)*
- [ ] **yarn 1.x** — después de instalar Node: `npm install -g yarn` (en Mac, si falla por permisos: `sudo npm install -g yarn`).
- [ ] **Supabase CLI** — para subir la base de datos: `npm install -g supabase` (Mac: `brew install supabase/tap/supabase`).
- [ ] **Git** — normalmente ya viene; verifica con `git --version`.
- [ ] **Cursor** — [cursor.com](https://cursor.com). Es el editor con IA que usarás para construir.

Verifica que todo quedó bien:

```bash
node --version   # v20.x o v22.x
yarn --version   # 1.22.x
git --version
```

<Callout type="warning" title="No mezcles npm y yarn">
Este repo usa **yarn** (hay un `yarn.lock`). Instala siempre con `yarn install`, nunca con `npm install`, o se te romperán las dependencias.
</Callout>

## 2 · Cuentas en servicios externos

Todas tienen versión gratuita. Crea las tres primeras hoy; las otras dos cuando lleguen en el temario.

- [ ] **[GitHub](https://github.com)** — ahí vas a crear tu repo en el Quick start.
- [ ] **[Supabase](https://supabase.com)** — base de datos + auth. Login con GitHub.
- [ ] **[Vercel](https://vercel.com)** — para publicar tu app. Login con GitHub.
- [ ] **[OpenAI](https://platform.openai.com)** — para las features de IA (Sem 3+). Opcional ahora.
- [ ] **[Resend](https://resend.com)** — para enviar emails (Sem 1+). Opcional ahora.

Detalle de qué es cada una: [Cuentas que necesitas](/docs/setup/cuentas).

<Checkpoint>
✅ Node, yarn y Cursor instalados.
✅ Cuentas de GitHub, Supabase y Vercel creadas.

¿Algo falla? → [Troubleshooting](/docs/troubleshooting/errores-comunes).
</Checkpoint>

## ¿Y ahora?

Con esto listo, ya puedes clonar el repo. El resto (instalar dependencias, configurar `.env.local`, correr el proyecto y hacer tu primer deploy) lo cubre el [Quick start](/docs/setup/quick-start) paso a paso.
