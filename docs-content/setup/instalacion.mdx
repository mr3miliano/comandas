Ya tienes el repo clonado. Antes de correr `yarn dev`, revisa esta lista. Marca cada casilla; cuando estén todas, estás listo.

Si prefieres el recorrido guiado paso a paso, ve a [Quick start](/docs/setup/quick-start). Esta página es la **lista de verificación** para tenerlo todo a la mano.

## 1 · Programas en tu computadora

<Callout type="info" title="¿Primera vez y no sabes cómo instalarlos?">
Los pasos exactos cambian según tu equipo. Sigue la guía por sistema operativo: [Prepara tu compu (Mac / Windows)](/docs/fundamentos/prepara-tu-compu).
</Callout>

- [ ] **Homebrew (solo Mac)** — el gestor de paquetes de Mac; facilita instalar todo lo demás:
  ```bash
  /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
  ```
- [ ] **Node 20 o 22 LTS** — [nodejs.org](https://nodejs.org) (versión **par**/LTS). En Mac con Homebrew: `brew install node`. El repo trae un `.nvmrc`: con [nvm](https://github.com/nvm-sh/nvm) corre `nvm use`.
- [ ] **yarn 1.x** — después de instalar Node: `npm install -g yarn` (en Mac, si falla por permisos: `sudo npm install -g yarn`).
- [ ] **Supabase CLI** — para subir la base de datos: `npm install -g supabase` (Mac: `brew install supabase/tap/supabase`).
- [ ] **Git** — normalmente ya viene; verifica con `git --version`.
- [ ] **Cursor** — [cursor.com](https://cursor.com). Es el editor con IA que usarás para construir.

Verifica que todo quedó bien:

```bash
node --version   # v20.x o v22.x
yarn --version   # 1.22.x
git --version
```

<Callout type="warning" title="No mezcles npm y yarn">
Este repo usa **yarn** (hay un `yarn.lock`). Instala siempre con `yarn install`, nunca con `npm install`, o se te romperán las dependencias.
</Callout>

## 2 · Cuentas en servicios externos

Todas tienen tier gratuito. Crea las tres primeras hoy; las otras dos cuando lleguen en el temario.

- [ ] **[GitHub](https://github.com)** — donde vive tu repo. (Ya lo tienes si clonaste.)
- [ ] **[Supabase](https://supabase.com)** — base de datos + auth. Login con GitHub.
- [ ] **[Vercel](https://vercel.com)** — para publicar tu app. Login con GitHub.
- [ ] **[OpenAI](https://platform.openai.com)** — para las features de IA (Sem 3+). Opcional ahora.
- [ ] **[Resend](https://resend.com)** — para enviar emails (Sem 1+). Opcional ahora.

Detalle de qué es cada una: [Cuentas que necesitas](/docs/setup/cuentas).

## 3 · Dependencias del proyecto

Desde la raíz del repo:

```bash
yarn install
```

- [ ] `yarn install` terminó sin errores (tarda 1–2 min la primera vez).

## 4 · Variables de entorno

```bash
cp web/.env.example web/.env.local
```

- [ ] Creaste `web/.env.local` copiando el ejemplo.
- [ ] Pegaste las 3 keys de Supabase (`NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`).

De dónde sale cada key: [Variables de entorno](/docs/setup/variables-de-entorno).

<Callout type="danger" title="Nunca subas .env.local a git">
Ya está en `.gitignore`. La `service_role` key es como la llave maestra de tu base de datos: no la compartas ni la pegues en el navegador.
</Callout>

## 5 · Corre el proyecto

```bash
yarn dev
```

- [ ] Abre [localhost:3000](http://localhost:3000) — ves tu landing.
- [ ] Abre [localhost:3000/docs](http://localhost:3000/docs) — ves estas docs.
