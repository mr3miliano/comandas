Antes de clonar VibeFast necesitas 5 programas en tu compu. Los pasos cambian entre **Mac** y **Windows** — ve directo a tu sección. Al final los dos caminos se juntan.

<Callout type="info" title="Los 5 programas (en ambos)">
1. **Node** (motor que corre el proyecto) · 2. **yarn** (baja las dependencias) · 3. **Git** (guarda y publica) · 4. **Supabase CLI** (sube tu base de datos) · 5. **Cursor** (el editor con IA). El navegador **Chrome** también ayuda.
</Callout>

## 🍎 En Mac

### 1. La terminal (ya la tienes)
Búscala con `Cmd + Espacio`, escribe **Terminal** y Enter. Es la app negra de comandos.

### 2. Homebrew (el gestor de paquetes de Mac)
Homebrew instala programas desde la terminal con un solo comando. Pega esto y sigue las instrucciones (te va a pedir tu contraseña de Mac):

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Al terminar, si te dice *"add Homebrew to your PATH"* con unos comandos, cópialos y córrelos tal cual. Verifica con `brew --version`.

<Callout type="info" title="¿No quieres usar Homebrew?">
Puedes instalar Node con el instalador **LTS** de [nodejs.org](https://nodejs.org) (`.pkg`) y saltarte este paso. Pero Homebrew te facilita instalar (y actualizar) todo lo demás.
</Callout>

### 3. Node
Con Homebrew:
```bash
brew install node
```
(O el instalador **LTS** de [nodejs.org](https://nodejs.org) si no usas brew.) Verifica con `node -v`.

### 4. yarn
```bash
npm install -g yarn
```

<Callout type="warning" title="¿Dice 'permission denied' / EACCES?">
Es por permisos. Reintenta con `sudo`:

```bash
sudo npm install -g yarn
```

Te va a pedir la **contraseña de tu Mac**. **No se ve nada mientras la escribes** (ni puntitos) — es normal por seguridad. Escríbela completa y dale Enter. (Alternativa sin sudo: `brew install yarn`.)
</Callout>

### 5. Git
Con Homebrew: `brew install git`. (Mac también lo ofrece vía *Command Line Tools*: corre `git --version` y acepta el botón que aparece, o `xcode-select --install`.)

### 6. Supabase CLI
Es la herramienta que sube tu base de datos a Supabase desde la terminal:
```bash
npm install -g supabase
```
(Alternativa con Homebrew: `brew install supabase/tap/supabase`.) Verifica con `supabase --version`.

### 7. Cursor
Descarga desde [cursor.com](https://cursor.com), abre el `.dmg` y arrastra Cursor a **Aplicaciones**.

## 🪟 En Windows

### 1. La terminal
Abre el menú Inicio, escribe **Terminal** (o **PowerShell**) y ábrelo. Es la ventana de comandos.

### 2. Node (versión LTS)
Descarga el instalador **LTS** desde [nodejs.org](https://nodejs.org) (archivo `.msi`), ábrelo y dale siguiente. **Deja marcada** la casilla que dice *"Add to PATH"* (viene marcada por defecto).

<Callout type="warning" title="Reinicia la terminal">
En Windows, después de instalar algo, **cierra y vuelve a abrir** la terminal para que reconozca el comando nuevo.
</Callout>

### 3. yarn
En PowerShell:
```powershell
npm install -g yarn
```

<Callout type="warning" title="¿PowerShell no deja ejecutar scripts?">
Si al correr `yarn` (o `npm`) sale algo como *"running scripts is disabled on this system"* / *"no se puede cargar el archivo yarn.ps1"*, corre **una vez** esto en PowerShell y responde `S` (o `Y`):

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

Luego cierra y reabre la terminal y reintenta.
</Callout>

### 4. Git
Descarga desde [git-scm.com](https://git-scm.com/download/win), abre el instalador y acepta **todas las opciones por defecto** (dale siguiente hasta el final). Esto también instala *Git Bash*, otra terminal útil.

### 5. Supabase CLI
Es la herramienta que sube tu base de datos a Supabase desde la terminal. En PowerShell:
```powershell
npm install -g supabase
```
Verifica con `supabase --version` (recuerda: cierra y reabre la terminal si no lo reconoce).

### 6. Cursor
Descarga el `.exe` desde [cursor.com](https://cursor.com) y ejecútalo. Sigue el instalador.

## ✅ Verifica (Mac y Windows)

Cierra y abre la terminal, y corre estos cuatro. Deberías ver un número de versión en cada uno:

```bash
node --version
yarn --version
git --version
supabase --version
```

<Callout type="danger" title="Si un comando dice 'command not found' / 'no se reconoce'">
Significa que ese programa no quedó instalado o la terminal no lo ve. En **Windows**, casi siempre se arregla cerrando y reabriendo la terminal. Si sigue, reinstala ese programa (en Windows, verifica *"Add to PATH"*). O pega el error en Cursor y pide ayuda — ver [Leer errores sin pánico](/docs/troubleshooting/leer-errores).
</Callout>

<Checkpoint>
✅ Tienes Node, yarn, Git, Supabase CLI y Cursor instalados.
✅ Los cuatro `--version` responden con un número.
✅ Sabes abrir tu terminal (Terminal en Mac, Terminal/PowerShell en Windows).
</Checkpoint>
