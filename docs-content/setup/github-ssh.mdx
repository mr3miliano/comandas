<StackBadge tech={["github", "git", "cursor"]} />

Cuando clonas por **HTTPS**, GitHub te pide un token cada vez que haces push — molesto y fácil de equivocar. Con **SSH** creas una **llave** en tu compu una sola vez, la registras en GitHub, y de ahí en adelante Cursor sube tu código **sin pedirte nada**. 🔑

<Callout type="info" title="¿Qué es una llave SSH? (en simple)">
Son **dos archivos**: una llave **privada** (se queda en tu compu, nunca la compartes) y una **pública** (esa sí la pegas en GitHub). GitHub reconoce que tu compu tiene la privada que corresponde y te deja entrar. Como una cerradura y su única llave.
</Callout>

<Callout type="info" title="Haz todo esto dentro de Cursor">
Todos los comandos de abajo se corren en la **terminal integrada de Cursor**: ábrela con **Terminal → New Terminal** o el atajo `` Ctrl+` `` (en Mac también `` Cmd+` ``). ¿Nunca la abriste? → [La terminal sin miedo](/docs/fundamentos/terminal). En **Windows**, si un comando falla en PowerShell, abre una terminal **Git Bash** con el botón `+` (flechita) del panel de terminal.
</Callout>

## 1 · Revisa si ya tienes una llave

Antes de crear una nueva, checa si ya existe. En la terminal integrada de Cursor (usa **Git Bash** si estás en Windows):

```bash
ls -al ~/.ssh
```

Si ves un archivo `id_ed25519.pub` (o `id_rsa.pub`), **ya tienes llave** → salta al **Paso 3**. Si dice que la carpeta no existe o está vacía, sigue al Paso 2.

## 2 · Crea tu llave

Corre esto **cambiando el correo** por el mismo que usas en GitHub:

```bash
ssh-keygen -t ed25519 -C "tu-correo@ejemplo.com"
```

Te va a preguntar 3 cosas — puedes darle **Enter** a las tres para aceptar lo default:

1. *"Enter file in which to save the key"* → Enter (usa la ubicación por defecto).
2. *"Enter passphrase"* → Enter para dejarla vacía (o pon una contraseña extra si quieres; te la pedirá al usar la llave).
3. *"Enter same passphrase again"* → Enter.

Listo: se crearon tu llave privada y tu llave pública en `~/.ssh/`.

<Callout type="warning" title="¿Windows viejo sin ssh-keygen?">
Si `ssh-keygen` no se reconoce en PowerShell, usa **Git Bash** (se instaló junto con Git). Ábrelo desde el menú Inicio y corre ahí el mismo comando.
</Callout>

## 3 · Copia tu llave **pública**

Ahora copia el contenido de la llave **pública** (la que termina en `.pub`) al portapapeles:

En **Mac**:
```bash
pbcopy < ~/.ssh/id_ed25519.pub
```

En **Windows** (PowerShell):
```powershell
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
```

En **Windows (Git Bash)** o si lo anterior falla:
```bash
cat ~/.ssh/id_ed25519.pub
```
…y selecciona y copia el texto que imprime (empieza con `ssh-ed25519` y termina con tu correo).

<Callout type="danger" title="La pública, NO la privada">
Solo copias el archivo que termina en **`.pub`**. El archivo **sin** `.pub` es tu llave privada — **nunca** la pegues en ningún lado ni la subas a GitHub.
</Callout>

## 4 · Pégala en GitHub

1. Entra a [github.com/settings/keys](https://github.com/settings/keys) (o: foto de perfil → **Settings** → **SSH and GPG keys**).
2. Botón **New SSH key**.
3. **Title**: un nombre para reconocer la compu (ej. `MacBook personal`).
4. **Key type**: deja **Authentication Key**.
5. **Key**: pega lo que copiaste en el Paso 3.
6. **Add SSH key**.

## 5 · Prueba la conexión

Confirma que GitHub te reconoce:

```bash
ssh -T git@github.com
```

La primera vez te pregunta *"Are you sure you want to continue connecting?"* → escribe **`yes`** y Enter. Si todo está bien, verás:

```text
Hi TU-USUARIO! You've successfully authenticated, but GitHub does not provide shell access.
```

Ese mensaje (aunque diga "does not provide shell access") **significa que funcionó**. 🎉

## 6 · Usa la URL SSH en tu repo

SSH funciona con la dirección **SSH** de tu repo (empieza con `git@github.com:`), no con la de HTTPS.

**Si vas a clonar de cero:** en tu fork de GitHub, botón verde **`Code`** → pestaña **SSH** → copia ese link. Se usa igual que en el [Quick start](/docs/setup/quick-start), solo cambia el link:

```bash
git clone git@github.com:TU-USUARIO/VibeFast.git
```

**Si ya clonaste por HTTPS** (y no quieres volver a bajar todo), solo cambia la dirección remota. Párate dentro de la carpeta del proyecto y corre:

```bash
git remote set-url origin git@github.com:TU-USUARIO/VibeFast.git
```

Verifica que quedó con `git remote -v` — debe mostrar la dirección que empieza con `git@github.com`.

<Callout type="info" title="A partir de aquí, todo es con clics">
Ya no vuelves a tocar esto. Tus commit y push desde Cursor (el ícono de **Source Control**) ahora suben solos, sin pedirte token. Repasa el flujo en [Git en 5 minutos](/docs/fundamentos/git-basico).
</Callout>

✅ Pegaste la llave **pública** en GitHub → SSH and GPG keys.
✅ `ssh -T git@github.com` te saludó con tu usuario.
✅ Tu repo usa la URL `git@github.com:...` (clon nuevo o `remote set-url`).
