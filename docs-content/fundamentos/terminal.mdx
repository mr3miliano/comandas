La **terminal** es una ventana donde le das órdenes a la compu escribiendo texto, en vez de hacer clic. Suena intimidante, pero para este curso solo usarás un puñado de comandos, casi siempre **copiar y pegar**.

<Callout type="info" title="No vas a escribir comandos raros de memoria">
En los tutoriales, los comandos vienen listos para copiar. Tu trabajo es pegarlos y apretar Enter.
</Callout>

## Abrir la terminal en Cursor

1. Abre Cursor con tu proyecto.
2. En el menú de arriba: **Terminal → New Terminal** (o el atajo `` Ctrl+` ``, esa tecla arriba del Tab).
3. Aparece un panel abajo con texto y un cursor parpadeando. Eso es la terminal.

<Callout type="info" title="¿Terminal de Cursor o del sistema?">
Da exactamente lo mismo usar la terminal integrada de Cursor que abrir una terminal externa (Terminal de Mac, PowerShell o CMD en Windows). Es la misma terminal, solo cambia la ventana donde la ves. Usa la de Cursor porque la tienes a la mano sin cambiar de ventana, pero si en algún momento se cierra o prefieres una aparte, puedes seguir los mismos comandos ahí sin problema.
</Callout>

## Correr tu primer comando

Escribe (o pega) esto y aprieta **Enter**:

```bash
node --version
```

Si te responde algo como `v22.x`, ¡funciona! Acabas de correr un comando.

<Callout type="warning" title="Copiar/pegar en la terminal">
En Mac es `Cmd+C` / `Cmd+V`. En Windows, a veces es clic derecho para pegar. Si `Ctrl+V` no pega, prueba `Ctrl+Shift+V`.
</Callout>

## ¿Se colgó o está trabajando?

La duda #1 de todos. Cuando corres algo como `yarn install` o `yarn dev`:

- **Está trabajando** si ves texto moviéndose, puntos, o una barra de progreso. **Espera.**
- **Terminó** cuando vuelve a aparecer el cursor listo para escribir.
- **`yarn dev` NO “termina”**: se queda corriendo a propósito (es tu servidor). Eso es normal — déjalo abierto.

## El botón de pánico: `Ctrl+C`

Si un comando se quedó pegado o quieres pararlo, haz clic en la terminal y aprieta **`Ctrl+C`**. Eso lo detiene. No rompe nada.

## Dos comandos que te ubican

```bash
pwd
```
Te dice **en qué carpeta estás parado** (path = ruta).

```bash
ls
```
**Lista** los archivos y carpetas de donde estás.

<Callout type="danger" title="Antes de correr un comando de internet">
Si copias un comando de un tutorial nuestro, tranquilo. Si lo copias de un sitio random, pídele a Cursor: “¿qué hace este comando? ¿es seguro?” antes de correrlo.
</Callout>
