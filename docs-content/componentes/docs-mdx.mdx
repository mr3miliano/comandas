Componentes disponibles dentro de cualquier archivo `.mdx` en `docs-content/`.

## PromptBox

Bloque copy-paste de un prompt para Cursor. Acepta una prop opcional `label` para ponerle un título al bloque (útil para distinguir varios prompts en la misma página).

```mdx
<PromptBox>
Tu prompt aquí.
</PromptBox>

<PromptBox label="Prompt para Cursor (ayuda con el copy)">
Tu prompt aquí.
</PromptBox>
```

## Callout

Avisos con 4 variantes: `info`, `warning`, `success`, `danger`.

```mdx
<Callout type="warning" title="Cuidado">
Mensaje.
</Callout>
```

## Checkpoint

Para el final de cada paso de tutorial.

```mdx
<Checkpoint>
Deberías ver X. ¿No lo ves? → [link](/docs/troubleshooting/errores-comunes).
</Checkpoint>
```

## StackBadge

Chips de tecnologías.

```mdx
<StackBadge tech={["next", "supabase", "openai"]} />
```

## VideoLoom

Embed de Loom. Props:

- `url` (opcional) — el link *share* de Loom (`https://www.loom.com/share/...`); el componente lo convierte a embed solo. Si **no** pasas `url`, en vez del video se muestra un placeholder que dice *"[ Video pendiente · grábalo con Loom ]"* — por eso es válido usarlo sin `url` mientras el video no existe.
- `title` (opcional) — título accesible del iframe. Default: `"Video del docente"`.

```mdx
<VideoLoom url="https://www.loom.com/share/XXXX" title="Mi video" />

<VideoLoom title="Video de la semana" />
```

El segundo (sin `url`) muestra el placeholder de video pendiente.
