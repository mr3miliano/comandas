## Qué vas a tener al final

## Concepto: structured outputs

En vez de pedirle al modelo "responde en formato JSON" (y esperar a que cumpla), le **das un schema con Zod** — o sea, un molde que describe exactamente qué campos debe traer la respuesta — y la API garantiza que el JSON cumple ese molde. Cambia todo: el modelo ya no es una caja negra, es una función con tipos.

```js
import { z } from "zod"
import { generateObject } from "@/lib/openai/structured"

const schema = z.object({
  resumen: z.string(),
  puntos_clave: z.array(z.string()),
  sentimiento: z.enum(["positivo", "neutral", "negativo"]),
})

const resultado = await generateObject(schema, "Analiza este review: ...")
// resultado.sentimiento === "positivo"  ← garantizado, tipado
```

## Pasos (preview)

1. Agregar `OPENAI_API_KEY` a `web/.env.local`. La key la creas en [platform.openai.com](https://platform.openai.com) → **API keys** — y necesitas saldo cargado en tu cuenta de OpenAI (ver [Setup → Cuentas](/docs/setup/cuentas)).
2. Importar el helper `generateObject`.
3. Definir el schema Zod de lo que quieres devolver.
4. Llamarla desde el backend — la IA (Cursor) elige el lugar correcto del código por ti.
5. Renderizar el resultado en una página.
