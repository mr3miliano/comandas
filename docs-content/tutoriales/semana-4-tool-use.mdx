## Qué vas a tener al final

- ✅ Una tool propia registrada en el registry.
- ✅ El modelo decide cuándo llamarla y con qué argumentos.
- ✅ Auditoría: cada tool call queda guardada en la tabla `tool_calls` (ya la creó tu `supabase db push` de la [Semana 2](/docs/tutoriales/semana-2-mvp)).
- ✅ Un flujo `recuperar → decidir → actuar` con [LangGraph](/docs/features/agentes-langgraph) — una librería para encadenar pasos de IA, como un diagrama de flujo que el modelo recorre.

## Concepto: tool use

Tool use es darle al modelo una lista de **funciones que puede invocar**. El modelo decide cuándo llamarlas y con qué argumentos; tú ejecutas la función y le devuelves el resultado. El loop sigue hasta que el modelo da una respuesta final.

VibeFast usa un **registry**: un catálogo central donde defines la tool una vez y queda disponible tanto para tool use (OpenAI) como para MCP (Sem 5). Más detalle en [Features → Tool use](/docs/features/tool-use).

## Pasos (preview)

1. Crear una tool en `lib/tools/examples/`.
2. Registrarla con `registerTool({ name, description, parameters, execute })`.
3. Probar desde `/chat` — la pantalla de chat de tu app, que se activa con `features.aiChat` en `web/config.js`.
4. Armar un flujo LangGraph que use 2+ tools.

## Entregable

Un flujo / acción de IA que ejecuta funciones reales de tu producto.
